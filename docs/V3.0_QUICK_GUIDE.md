## 1. 多通道采集

### 1.1 添加和配置通道

#### 通过 UI 操作

1. **打开通道管理面板**
   - 菜单: `视图` → `通道管理面板`
   - 快捷键: `Ctrl+M`

2. **添加通道**
   - 点击 `+ 添加通道` 按钮
   - 系统自动分配通道 ID

3. **配置通道**
   - **标签**: 双击标签列编辑通道名称
   - **颜色**: 点击颜色按钮选择显示颜色
   - **可见性**: 勾选复选框显示/隐藏通道
   - **分组**: 下拉选择或输入新分组名

4. **保存配置**
   - 配置自动保存到 `~/.xgen-waveform-viewer/channel_config.json`

#### 通过代码操作

```python
from xgen_waveform_viewer.multi_channel import MultiChannelManager

# 创建管理器
manager = MultiChannelManager()

# 添加多个通道
channels = [
    {"id": 0, "label": "温度传感器", "color": "#ff0000", "group": "环境监测"},
    {"id": 1, "label": "湿度传感器", "color": "#0000ff", "group": "环境监测"},
    {"id": 2, "label": "光照传感器", "color": "#00ff00", "group": "环境监测"},
    {"id": 3, "label": "压力传感器", "color": "#ffff00", "group": "压力监测"},
]

for ch in channels:
    manager.add_channel(
        channel_id=ch["id"],
        label=ch["label"],
        color=ch["color"]
    )
    manager.update_channel_config(ch["id"], group=ch["group"])

# 添加数据
import numpy as np

for i in range(100):
    # 模拟不同通道的数据
    for ch_id in range(4):
        samples = np.random.randint(0, 4096, size=256, dtype=np.uint16)
        manager.append_data(
            channel_id=ch_id,
            samples=samples,
            seq=i,
            timestamp=i * 0.01
        )
```

### 1.2 多通道显示模式

#### 叠加模式（Overlay）
所有通道显示在同一个坐标系中，适合比较波形。

```python
# 设置所有通道的 Y 偏移为 0
for ch_id in range(4):
    manager.update_channel_config(ch_id, y_offset=0.0)
```

#### 堆叠模式（Stack）
每个通道有独立的垂直位置，避免重叠。

```python
# 为每个通道设置不同的 Y 偏移
offset_step = 5000  # 每通道间隔 5000 单位
for ch_id in range(4):
    manager.update_channel_config(ch_id, y_offset=ch_id * offset_step)
```

## 2. 自定义协议配置

### 2.1 使用预设协议

#### Binary V2（默认）
无需配置，开箱即用。

```python
from xgen_waveform_viewer.protocol import ProtocolFactory, ProtocolType

parser = ProtocolFactory.create_parser(ProtocolType.BINARY_V2)
```

#### ASCII 文本协议
适用于调试和简单设备。

```python
config = {
    "line_terminator": b"\n",
    "delimiter": ",",
    "has_channel_prefix": False,
}

parser = ProtocolFactory.create_parser(ProtocolType.ASCII, config)
```

### 2.2 创建自定义二进制协议

#### 步骤 1: 定义帧格式

假设您的设备使用以下帧格式：
```
[0xAA][0x55] - 同步头
[CH:1B] - 通道 ID
[LEN:2B] - 数据长度
[DATA:LEN×2B] - 采样数据
[CHK:1B] - 简单校验和
```

#### 步骤 2: 创建配置文件

创建 `my_protocol.json`:

```json
{
  "protocol_type": "binary_custom",
  "frame_format": {
    "sync_bytes": "AA55",
    "meta_size": 3,
    "has_crc": false,
    "sample_size": 2,
    "endianness": "big",
    "has_sequence": false,
    "has_channel_id": true,
    "channel_id_offset": 0,
    "sample_count_offset": 1,
    "max_samples": 512
  }
}
```

#### 步骤 3: 加载和使用

```python
from xgen_waveform_viewer.protocol import ProtocolFactory

# 从文件加载
parser = ProtocolFactory.load_from_file("my_protocol.json")

# 在串口读取中使用
# 参见 serial_reader.py，替换解析器即可
```

### 2.3 通过 UI 配置协议

1. **打开协议配置面板**
   - 菜单: `工具` → `协议配置`

2. **选择协议类型**
   - Binary V2: 默认协议
   - Binary Custom: 自定义二进制
   - ASCII Text: 文本协议

3. **配置参数**
   - 根据所选协议类型填写参数
   - 参考帮助文本和示例

4. **测试和应用**
   - 点击 `应用配置` 测试
   - 点击 `导出配置` 保存为文件

### 2.4 ASCII 协议示例

#### 简单 CSV 格式
```
1234,5678,9012,3456
2345,6789,0123,4567
```

配置:
```python
config = {
    "line_terminator": b"\n",
    "delimiter": ",",
    "has_channel_prefix": False,
}
```

#### 带通道标识格式
```
#CH0:1234,5678,9012
#CH1:2345,6789,0123
#CH2:3456,7890,1234
```

配置:
```python
config = {
    "line_terminator": b"\n",
    "delimiter": ",",
    "has_channel_prefix": True,
    "channel_prefix": "#CH",
}
```

## 3. 固件配置

### 3.1 获取固件信息

#### 通过 UI
1. 连接设备
2. 打开 `工具` → `固件配置`
3. 点击 `获取版本信息`

#### 通过代码
```python
from xgen_waveform_viewer.firmware_config import FirmwareConfigManager

# 假设已有串口连接
manager = FirmwareConfigManager(serial_port)

# 获取版本
manager.get_firmware_version()

# 监听信号
def on_version_received(version):
    print(f"固件版本: {version}")
    print(f"  主版本: {version.major}")
    print(f"  次版本: {version.minor}")
    print(f"  补丁: {version.patch}")
    print(f"  构建: {version.build}")
    print(f"  Commit: {version.commit_hash}")
    print(f"  日期: {version.build_date}")

manager.version_received.connect(on_version_received)

# 等待响应
manager.process_response(timeout=2.0)
```

### 3.2 配置固件参数

#### 设置采样率
```python
# 设置为 50 kHz
manager.set_sample_rate(50000)
manager.process_response()
```

#### 设置帧长
```python
# 每帧 512 个采样点
manager.set_frame_length(512)
manager.process_response()
```

#### 批量配置
```python
from xgen_waveform_viewer.firmware_config import FirmwareConfig

config = FirmwareConfig(
    sample_rate=100000,      # 100 kHz
    frame_length=1024,       # 1024 samples/frame
    channel_count=4,         # 4 通道
    channel_mask=0x0F,       # 启用通道 0-3
    adc_resolution=12,       # 12 位 ADC
    adc_vref=3.3,           # 3.3V 参考电压
    trigger_enabled=True,    # 启用触发
    trigger_level=2048,      # 触发电平
    trigger_edge="rising",   # 上升沿触发
)

manager.set_firmware_config(config)
manager.process_response()
```

### 3.3 OTA 固件更新

#### 准备固件文件
确保您有正确的固件文件（.bin 格式）。

#### 通过 UI 更新
1. 打开 `工具` → `固件配置`
2. 滚动到 `固件更新 (OTA)` 部分
3. 点击 `选择固件文件`
4. 选择 .bin 文件
5. 点击 `开始更新`
6. **重要**: 更新过程中请勿断开连接

#### 通过代码更新
```python
# 读取固件文件
with open("firmware_v3.0.bin", "rb") as f:
    firmware_data = f.read()

print(f"固件大小: {len(firmware_data)} bytes")

# 定义进度回调
def on_progress(current, total):
    progress = current * 100 / total
    print(f"更新进度: {progress:.1f}% ({current}/{total} bytes)")

def on_completed(success):
    if success:
        print("✓ OTA 更新成功！设备将重启...")
    else:
        print("✗ OTA 更新失败")

# 连接信号
manager.ota_progress.connect(on_progress)
manager.ota_completed.connect(on_completed)

# 开始更新
manager.ota_update(firmware_data)
```

#### OTA 更新注意事项
- ⚠️ **更新前备份**: 确保有固件备份
- ⚠️ **不要断电**: 更新过程中保持设备供电
- ⚠️ **不要断开**: 不要断开串口连接
- ⚠️ **波特率影响**: 更新时间取决于波特率（建议 921600+）
- ✅ **自动验证**: 系统自动验证固件完整性
- ✅ **失败恢复**: 更新失败设备会自动回滚

### 3.4 兼容性检查

```python
from xgen_waveform_viewer.firmware_config import (
    FirmwareVersion, check_firmware_compatibility
)

# 当前固件版本
current_version = FirmwareVersion(3, 0, 0)

# 最低要求版本
min_required = FirmwareVersion(2, 4, 0)

# 检查兼容性
is_compatible, message = check_firmware_compatibility(
    current_version,
    min_required
)

if is_compatible:
    print(f"✓ {message}")
else:
    print(f"✗ {message}")
    print("请更新固件！")
```

## 4. 常见问题

### Q1: 如何同时显示多个通道？

**A**: 
1. 添加所需的通道（通道管理面板）
2. 确保所有通道的 `可见` 复选框已勾选
3. 调整各通道的颜色以便区分
4. 可选：设置 Y 偏移避免波形重叠

### Q2: 自定义协议不工作怎么办？

**A**: 
1. 检查同步头是否正确（十六进制格式）
2. 验证元数据大小和偏移量
3. 确认字节序（little/big endian）
4. 使用日志查看解析错误详情
5. 尝试导出配置并手动检查 JSON 格式

### Q3: OTA 更新失败如何恢复？

**A**: 
- 设备通常会自动回滚到原固件
- 如果设备无响应：
  1. 断开连接
  2. 重新上电
  3. 尝试进入 Bootloader 模式（参考设备文档）
  4. 使用专用烧录工具恢复固件

### Q4: 多通道采集性能问题？

**A**: 
- **降低采样率**: 根据实际需求调整
- **减少帧长**: 使用较小的帧长
- **隐藏不需要的通道**: 仅显示必要通道
- **限制缓冲区**: 在设置中调整缓冲区大小
- **升级硬件**: 使用更快的 CPU 或增加内存

### Q5: 如何保存多通道配置？

**A**: 
通道配置自动保存到配置文件：
- Windows: `%USERPROFILE%\.xgen-waveform-viewer\channel_config.json`
- Linux/Mac: `~/.xgen-waveform-viewer/channel_config.json`

手动导出：
```python
# 获取配置
config = manager.save_config()

# 保存到文件
import json
with open("my_channels.json", "w") as f:
    json.dump(config, f, indent=2)

# 加载配置
with open("my_channels.json", "r") as f:
    config = json.load(f)
manager.load_config(config)
```

### Q6: 固件不支持配置命令怎么办？

**A**: 
- V3.0 的固件配置功能需要固件版本 >= 3.0.0
- 旧版本固件只能通过重新编译和烧录修改参数
- 建议升级到 V3.0+ 固件以使用新功能

### Q7: ASCII 协议数据解析错误？

**A**: 
1. 检查行终止符（\n, \r\n, \r）
2. 确认分隔符（逗号、制表符等）
3. 验证数据格式（整数、浮点数）
4. 检查通道前缀格式
5. 查看日志中的解析错误详情
