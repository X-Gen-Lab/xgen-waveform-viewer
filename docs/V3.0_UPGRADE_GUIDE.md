## 升级前准备

### 1. 备份现有配置

V3.0 会自动迁移配置，但建议备份以防万一：

**Windows:**
```cmd
xcopy "%USERPROFILE%\.xgen-waveform-viewer" "%USERPROFILE%\.xgen-waveform-viewer.backup" /E /I
```

**Linux/Mac:**
```bash
cp -r ~/.xgen-waveform-viewer ~/.xgen-waveform-viewer.backup
```

### 2. 保存录制文件

确保重要的录制文件已妥善保存：
- BIN 文件：完整保留
- CSV 文件：完整保留
- 导出的图片和报告：备份

### 3. 检查系统要求

V3.0 系统要求与 V2.x 相同：
- **Python**: 3.10 或更高版本
- **操作系统**: Windows 10+, Linux, macOS
- **内存**: 建议 4GB 或更多
- **磁盘空间**: 至少 100MB 可用空间

### 4. 记录当前配置

记录以下配置以便验证迁移：
- 串口设置（端口、波特率）
- 显示设置（X/Y 轴范围）
- 录制设置（格式、保存目录）
- 主题选择

## 固件升级

### 检查固件版本

V3.0 新功能需要固件版本 >= 3.0.0。

1. 连接设备
2. 打开 `工具` → `固件配置`
3. 点击 `获取版本信息`

如果固件版本 < 3.0.0，建议升级以使用完整功能。

### 固件升级步骤

#### ⚠️ 重要警告
- 更新前备份当前固件
- 更新过程中不要断电或断开连接
- 准备固件文件（.bin 格式）

#### 升级流程

1. **准备固件文件**
   - 从官方渠道获取 V3.0 固件
   - 确认文件完整性（MD5/SHA256）

2. **启动 OTA 更新**
   - 打开 `工具` → `固件配置`
   - 滚动到 `固件更新 (OTA)` 部分
   - 点击 `选择固件文件`
   - 选择 .bin 文件

3. **执行更新**
   - 点击 `开始更新`
   - 观察进度条
   - 等待完成（约 10-30 秒）

4. **验证更新**
   - 设备自动重启
   - 重新连接
   - 点击 `获取版本信息`
   - 确认版本为 3.0.0+

#### 更新失败处理

如果更新失败：
1. 设备通常会自动回滚到原固件
2. 重新上电设备
3. 尝试使用专用烧录工具恢复
4. 联系技术支持

### 无固件升级的兼容性

如果保持使用 V2.x 固件：
- ✅ 基础采集功能正常
- ✅ 单通道模式正常
- ✅ 录制和回放正常
- ❌ 无法使用多通道功能
- ❌ 无法远程配置参数
- ❌ 无法使用 OTA 更新

## 新功能使用

### 1. 多通道功能

#### 启用多通道
1. 确保固件版本 >= 3.0.0
2. 打开 `视图` → `通道管理面板`
3. 点击 `+ 添加通道`
4. 配置通道标签和颜色

#### 迁移单通道项目
单通道项目自动使用通道 0，无需修改。

### 2. 自定义协议

#### 使用预设协议
1. 打开 `工具` → `协议配置`
2. 选择协议类型（Binary V2 / Binary Custom / ASCII）
3. 点击 `应用配置`

#### 导入自定义协议
1. 准备协议配置 JSON 文件
2. 点击 `导入配置`
3. 选择配置文件
4. 点击 `应用配置`

#### V2.x 用户注意
默认使用 Binary V2 协议，与 V2.x 完全兼容。

### 3. 固件配置

#### 基本配置
1. 打开 `工具` → `固件配置`
2. 点击 `读取配置` 查看当前参数
3. 修改参数（采样率、帧长等）
4. 点击 `写入配置`

#### 高级功能
- 触发配置：设置硬件触发参数
- 通道配置：启用/禁用特定通道
- ADC 配置：调整分辨率和参考电压

## 常见问题

### Q1: 升级后无法启动？

**A**: 
1. 检查 Python 版本（需要 3.10+）
2. 重新安装依赖：
   ```bash
   pip install --force-reinstall xgen-waveform-viewer
   ```
3. 删除配置文件重新初始化：
   ```bash
   rm -rf ~/.xgen-waveform-viewer
   ```

### Q2: 配置没有自动迁移？

**A**:
1. 确认 V2.x 配置文件存在
2. 手动复制配置：
   ```bash
   cp ~/.xgen-waveform-viewer.backup/config.json ~/.xgen-waveform-viewer/
   ```
3. 重启应用触发迁移

### Q3: 旧录制文件无法播放？

**A**:
V3.0 完全兼容 V2.x 录制文件。如果无法播放：
1. 检查文件完整性（未损坏）
2. 确认文件格式（BIN/CSV）
3. 查看日志文件排查错误

### Q4: 多通道功能不可用？

**A**:
多通道需要固件支持：
1. 检查固件版本（需要 >= 3.0.0）
2. 升级固件到 V3.0
3. 或使用单通道模式（与 V2.x 相同）

### Q5: 自定义协议不工作？

**A**:
1. 验证 JSON 配置文件格式
2. 检查同步头和帧格式定义
3. 使用示例配置测试
4. 查看日志文件获取错误详情

### Q6: OTA 更新失败？

**A**:
1. 检查串口连接稳定性
2. 提高波特率重试（建议 921600+）
3. 使用专用烧录工具作为备选
4. 确认固件文件正确（.bin 格式，未损坏）

## 回滚方案

如果需要回滚到 V2.x：

### 1. 卸载 V3.0

```bash
pip uninstall xgen-waveform-viewer
```

### 2. 安装 V2.4.1

```bash
pip install xgen-waveform-viewer==2.4.1
```

### 3. 恢复配置

```bash
# Windows
xcopy "%USERPROFILE%\.xgen-waveform-viewer.backup" "%USERPROFILE%\.xgen-waveform-viewer" /E /I /Y

# Linux/Mac
cp -rf ~/.xgen-waveform-viewer.backup/* ~/.xgen-waveform-viewer/
```

### 4. 验证

启动应用，确认版本和功能正常。
