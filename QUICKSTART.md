## 安装

### 方式一：下载可执行文件（推荐）
1. 访问 [Releases](https://github.com/X-Gen-Lab/xgen-waveform-viewer/releases) 页面
2. 下载最新版本的 `xgen-waveform-viewer-VX.X-windows-x64.zip`
3. 解压到任意目录
4. 双击 `xgen-waveform-viewer.exe` 运行

### 方式二：从源码运行
```powershell
# 克隆仓库
git clone https://github.com/X-Gen-Lab/xgen-waveform-viewer.git
cd xgen-waveform-viewer

# 创建虚拟环境
python -m venv .venv
.\.venv\Scripts\Activate.ps1

# 安装依赖
pip install -r requirements.txt

# 运行程序
python main.py
```

## 首次启动

首次启动时，您将看到：
- **深色主题**的界面（可在 View > Theme 菜单切换）
- **空白波形区域**，显示"等待有效 ADC 帧"
- **工具栏**包含串口配置和控制按钮
- **状态栏**显示"Disconnected"

## 连接设备

### 1. 选择串口
- 点击 **Port** 下拉框，选择您的串口设备（如 `COM3`）
- 如果没有看到设备，请：
  - 检查设备是否已连接
  - 点击 **Refresh** 按钮刷新列表
  - 确认驱动程序已正确安装

### 2. 配置串口参数
默认配置通常已足够，如需修改：
- **Baud**: 波特率（默认 460800）
- **Data**: 数据位（默认 8）
- **Stop**: 停止位（默认 1）
- **Parity**: 校验位（默认 None）

### 3. 连接
点击 **Connect** 按钮（或按 `C` 键）

✅ **成功标志**：
- 按钮变为 "Disconnect"
- 状态栏显示 "Connected: COMx @ xxxxx"
- 开始接收数据时，波形区域显示实时波形

❌ **连接失败**：
- 检查串口是否被其他程序占用
- 确认设备已上电并正常工作
- 尝试降低波特率

## 查看波形

### 基本操作

**鼠标操作**：
- 🖱️ **左键拖拽**：平移视图（X/Y 轴）
- 🖱️ **滚轮**：缩放（Ctrl+滚轮仅缩放 Y 轴）
- 🖱️ **双击**：恢复自动滚动

**键盘快捷键**：
- `Space`：恢复自动滚动
- `F`：显示缓冲区全部数据
- `+` / `-`：X 轴放大/缩小
- `Y`：切换 Y 轴自动/手动模式

### X 轴控制

**窗口宽度**：
- 使用 **X:** 下拉框或数值框调整显示时间跨度
- 预设值：10ms ~ 6小时
- 自定义：直接输入数值

**缩放与平移**：
- `+` / `-`：围绕视图中心缩放
- **All** 按钮：显示全部缓冲区数据
- **Follow** 按钮：恢复自动滚动到最新数据

**缓冲区大小**：
- **Buf:** 数值框设置最大缓冲点数（单位：百万点）
- 默认 10Mpts，可根据内存情况调整

### Y 轴控制

**自动模式**（默认）：
- Y 轴自动缩放以显示当前可见数据
- 点击 **Auto** 按钮或按 `Y` 切换到手动模式

**手动模式**：
- **Min** / **Max** 数值框：精确设置 Y 轴范围
- **Full** 按钮：显示 ADC 全范围（0-4095）
- **Center** 按钮：以当前数据为中心，保留余量

## 数据录制

### 开始录制

1. **选择格式**：
   - **BIN**：高效二进制格式（推荐，文件更小）
   - **CSV**：文本格式（便于第三方工具处理）

2. **设置保存路径**（可选）：
   - 点击 **Path...** 按钮设置保存目录
   - 不设置则每次录制时弹出文件对话框

3. **开始录制**：
   - 点击 **Record** 按钮（或按 `R` 键）
   - 按钮变为红色 "Stop"
   - 状态栏显示 "REC [格式]: 文件名"

4. **停止录制**：
   - 再次点击 **Stop** 按钮（或按 `R` 键）
   - 状态栏显示录制统计信息

### 保存当前缓冲区

不想录制，只想保存当前显示的数据？

- **Save Buffer** 按钮（或 `Ctrl+S`）：保存为 BIN 格式
- **Export CSV** 按钮（或 `Ctrl+E`）：导出为 CSV

### 数据完整性

录制文件会附带 `.meta.json` 元数据文件，包含：
- 录制统计（帧数、采样点数、时长）
- 数据完整性指标（CRC 错误、序列号间隙等）
- 串口统计信息

⚠️ 如果检测到数据丢失，会弹出警告对话框。

## 故障排除

### 问题：看不到波形

**可能原因**：
1. ❌ 设备未发送数据
   - 检查设备是否正常工作
   - 确认波特率设置正确
2. ❌ 帧格式不匹配
   - 确认固件使用相同的帧协议
   - 查看状态栏的 "CRC Error" 和 "Resync" 计数
3. ❌ 数据被过滤
   - 点击 **All** 按钮查看全部数据
   - 检查 Y 轴范围设置

### 问题：波形卡顿或延迟

**解决方法**：
1. 降低波特率
2. 减小 X 轴窗口宽度
3. 降低缓冲区大小
4. 关闭其他占用 CPU 的程序

### 问题：串口连接失败

**解决方法**：
1. 检查串口是否被占用（关闭其他串口工具）
2. 确认驱动程序已安装
3. 尝试其他串口或 USB 端口
4. 以管理员权限运行程序（Windows）

### 问题：CRC 错误或序列号间隙

**原因**：
- 串口缓冲区溢出（波特率过高）
- 电磁干扰
- 线缆质量问题

**解决方法**：
1. 启用 **Auto Reconnect**（Connection > Auto Reconnect）
2. 使用屏蔽线缆
3. 降低波特率
4. 减小每帧采样点数（固件配置）
